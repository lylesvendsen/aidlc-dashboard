# B8 - AIDLC Live Session
# Version: 1.0
# Part of: AIDLC Dashboard
# Updated: March 2026

## What this unit delivers
A Git-based collaborative session feature that lets multiple developers participate in the same AIDLC run. The session host runs specs normally — after each SP passes, changes are committed and pushed to a shared session branch. Participants run a watcher CLI that polls the branch, pulls changes to their local appDir, and triggers local validation. All participants see live session status in the dashboard.

## Context
- Requires B0 (Run page) complete
- Requires the target project to have a Git repo initialised in appDir
- No prior collaboration units required
- Session coordination is Git-only — no WebSockets, no server state

## Architecture principles
- Git is the transport layer — no custom sync protocol
- The runner commits after each SP pass using an optional session branch name
- Participants run a lightweight CLI watcher (aidlc-watch) that polls and applies
- Dashboard shows session status as a read-only panel — who is in the session, last sync time
- Session branch naming: aidlc/session-{sessionId} (e.g. aidlc/session-abc123)
- Watcher writes nothing to the dashboard's own files — it only touches the participant's appDir
- Session ID is a short random token generated by the host and shared out-of-band (Slack, etc.)
- All session config lives in .env.local alongside existing AIDLC vars

## Environment variables
```
AIDLC_SESSION_ID=abc123
AIDLC_SESSION_BRANCH=aidlc/session-abc123
AIDLC_SESSION_REMOTE=origin
AIDLC_SESSION_AUTO_COMMIT=true
AIDLC_SESSION_POLL_INTERVAL_MS=5000
```

## Git commit convention
Each SP commit message follows:
  aidlc(session-abc123): SP-01 Monorepo scaffold [passed]

Tags are applied at unit completion:
  aidlc/session-abc123/A0-complete

## Watcher CLI
Participants run from the aidlc-dashboard root:
  npx tsx scripts/aidlc-watch.ts --project mn9ba6erhzo5

The watcher:
1. Reads project config to get appDir and session settings
2. Polls AIDLC_SESSION_BRANCH every AIDLC_SESSION_POLL_INTERVAL_MS ms
3. On new commits: git pull --ff-only, then runs afterEachSubPrompt validation locally
4. Prints results to stdout with timestamp
5. Optionally posts results back to a session log file in logsDir

## Session status in dashboard
A collapsible "Session" panel in the run page shows:
- Session ID and branch name
- Last commit synced (SHA + SP name + timestamp)
- Participant count (read from a session-participants.json file in logsDir)
- Each participant's last validation result

Participants register by writing an entry to logsDir/session-{id}-participants.json
when the watcher starts.

### SP-03: Watcher CLI script
Create the participant watcher script that polls the session branch and runs
local validation when new commits appear.

Files to create:
  scripts/aidlc-watch.ts

Behaviour:
  1. Parse --project flag to load project config
  2. Read session config — exit with error if not configured
  3. Register self in session participants file
  4. Poll loop every AIDLC_SESSION_POLL_INTERVAL_MS:
     a. Call pullLatest()
     b. If newCommits > 0: print commit info, run each afterEachSubPrompt command
     c. Print pass/fail for each validation command with timestamp
  5. On SIGINT: print summary and exit cleanly

Output format:
  [14:23:01] Polling aidlc/session-abc123...
  [14:23:06] 1 new commit: SP-01 Monorepo scaffold [passed]
  [14:23:06] Running validation...
  [14:23:08]   npm run typecheck  PASS
  [14:23:10]   npm run lint       PASS
  [14:23:10] All checks passed. Waiting for next commit...

Acceptance:
  SP03-01  scripts/aidlc-watch.ts exists
  SP03-02  Script reads project config by --project flag
  SP03-03  Script exits with clear error if no session config found
  SP03-04  Script registers participant on start
  SP03-05  npm run typecheck passes
